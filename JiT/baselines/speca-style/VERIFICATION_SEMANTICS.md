# JiT verification semantics

## Local two-branch check

On a Taylor NFE with `check=true`, blocks before the selected verify layer execute the ordinary draft path. Let the resulting input to the selected block be `x_verify_in`; it is already a speculative-prefix state. The port evaluates two branches from that identical tensor and current conditioning:

```text
x_pred  = draft_block(x_verify_in, fresh current gates, Taylor factors)
x_exact = exact_block(x_verify_in, fresh modulation/norm/attention/MLP/gates)
error   = official_metric(x_pred, x_exact)
main path continues with x_pred
```

The verifier default is `verify_layer=-1`, resolved at runtime as `len(model.blocks)-1`. For JiT-B/16 that is block 11, not the released DiT-XL/2 literal 27. An explicit other layer is an ablation and must be recorded.

## What is and is not exact

The verifier block itself is exact conditional on `x_verify_in`; its prefix is not exact. It compares complete post-residual block outputs. It is not a comparison of full-model outputs, isolated attention tensors, output-head tensors, or a hidden state generated by a fully exact prefix.

The exact branch is metric-only:

- it never replaces `x_pred`;
- it never updates attention/MLP Taylor factors;
- it never rewinds Heun state;
- it never reruns preceding blocks;
- failure is written at `end_nfe` and can force Full only at the next `begin_nfe`.

Final norm, projection, context removal, and unpatchify always run fresh on the main `x_pred` path.

## Tokens and JiT CFG streams

Main `verification_token_scope=all_tokens`. JiT inserts class-context tokens at `in_context_start` and retains them through the final block, so final-block error includes image and context tokens. `image_tokens_only` is an explicit diagnostic ablation, not released-code-faithful main behavior.

Conditional and unconditional streams use independent speculative prefixes and payloads. The main scalar is mathematically equivalent to computing the official metric on:

```text
cat([pred_cond, pred_uncond], dim=0)
cat([exact_cond, exact_uncond], dim=0)
```

Streaming sums are allowed to avoid materializing the concatenation, but tests must match real concatenation for L1, L2, relative L1, relative L2, and cosine. A missing stream is a fail-fast error. Both streams continue with their own draft output.

## Error and synchronization

Main metric:

```text
relative_l1 = mean(abs(pred-exact)/(abs(exact)+1e-10))
failure     = current_error > current_threshold
```

The reduction covers batch, tokens, and channels. Equality passes. Conversion to a Python scalar is part of released behavior and its synchronization time must be included in latency and verification-overhead accounting.

## Clone optimization

Released code clones block inputs broadly. This port retains only the input needed at the verify layer when `check=true`; other blocks and unchecked Taylor NFEs do not clone for verification. This removes dead temporary work without altering either branch. CPU mock tests establish same-input branching, all-token reduction, no writeback, no history update, and previous-error timing; real model/block parity and CUDA memory remain deferred.
